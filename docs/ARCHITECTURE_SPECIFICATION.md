## 1. Executive Summary

### 1.1 Purpose

BidDeed.AI is an **Agentic AI ecosystem** designed to provide intelligent foreclosure auction analysis for real estate investors. The platform processes properties through a proprietary 12-stage methodology called **The Everest Ascentâ„¢**, delivering actionable BID/REVIEW/SKIP recommendations with ML-powered confidence scores.

### 1.2 Business Context

| Metric | Value |
|--------|-------|
| **Target Market** | Brevard County, Florida foreclosure auctions |
| **Annual Value Created** | $300-400K (avoided losses + captured deals) |
| **System Cost** | $3.3K/year |
| **ROI** | 100x |
| **Primary User** | Solo investor with 10+ years foreclosure experience |

### 1.3 Architectural Principles

1. **Agentic-First Design** - Autonomous agents over monolithic services
2. **Cost-Optimized AI** - Smart Router for 40-55% free LLM usage
3. **Fail-Safe Operations** - Checkpointing at every pipeline stage
4. **Data Integrity** - Layer 8 IP protection for proprietary formulas
5. **Minimal Human-in-Loop** - Autonomous execution with exception handling

### 1.4 Technology Stack Summary

| Layer | Technologies |
|-------|--------------|
| **Frontend** | Next.js 15, React 18, Tailwind CSS, assistant-ui |
| **Orchestration** | LangGraph, Python 3.11 |
| **AI/ML** | XGBoost, Claude Sonnet 4, Gemini 2.5 Flash |
| **Database** | Supabase (PostgreSQL 15) |
| **Infrastructure** | GitHub Actions, Cloudflare Pages, Modal |
| **Monitoring** | Supabase Logs, GitHub Actions Insights |

### 2.2 Architectural Style

BidDeed.AI employs a **Hybrid Architecture** combining:

| Pattern | Application | Rationale |
|---------|-------------|-----------|
| **Agentic AI** | Pipeline orchestration | Autonomous multi-stage processing |
| **Microservices** | Scrapers, ML, Reports | Independent deployment and scaling |
| **Event-Driven** | WebSocket updates | Real-time UI state synchronization |
| **Serverless** | GitHub Actions, Cloudflare | Cost optimization, auto-scaling |
| **CQRS** | Read/Write separation | Optimized query patterns |

### 2.3 Key Architectural Decisions

| Decision | Choice | Alternatives Considered | Rationale |
|----------|--------|------------------------|-----------|
| **Orchestration** | LangGraph | Temporal, Airflow | Native AI agent support, checkpointing |
| **LLM Strategy** | Smart Router | Single provider | 40-55% cost reduction |
| **Database** | Supabase | Firebase, PlanetScale | PostgreSQL features, realtime, auth |
| **Hosting** | Cloudflare Pages | Vercel, Netlify | Edge performance, cost |
| **Automation** | GitHub Actions | Jenkins, CircleCI | Native integration, free tier |
